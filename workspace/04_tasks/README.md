# 04_tasks — 任务看板目录

> 这个目录是 ClawRPA 项目的任务管理中心。
> 每个任务按标准结构组织，所有进度信息在这里可追溯。

---

## 目录结构

```
04_tasks/
├── README.md              # 本文件 — 结构设计要求
├── board.md               # 状态看板 — 所有任务一览
├── T0XX_NAME/
│   ├── plan.md            # 方案说明（静态：目标、架构、方案）
│   ├── board.md           # 状态跟踪（动态：子任务、进度）
│   ├── memory.md          # 过程记忆（增长型：踩坑、试错）
│   └── files/             # 产出物目录
└── ...
```

## 三文件 + 一目录的定位

| 文件 | 回答什么问题 | 写给谁 | 更新频率 |
|------|-------------|--------|----------|
| `plan.md` | 这是什么、为什么要做、怎么做 | 新加入的人快速了解 | 低，方案定了就不动 |
| `board.md` | 现在到什么阶段了、还剩什么 | 自己每天看进度 | 高，每次进展都要改 |
| `memory.md` | 试过什么、怎么失败、学到了什么 | 未来的自己复盘 | 中，每次踩坑都要记 |
| `files/` | 产出物放哪里 | 自己和队友找产出 | 随产出增加 |

## 任务状态标识

```
⬜ todo       → 未开始
🟡 doing      → 进行中（有进展和阻塞项）
🔵 verifying  → 验证中（实现完成，待验证结果）
✅ done       → 已完成
⏸️ archived   → 暂停/归档（附原因）
❌ cancelled  → 取消（附原因）
```

> 状态标识只写在 T0XX/board.md 顶部 header。

## 文件模板

### plan.md

```markdown
# T0XX — 任务名称

> **方案：** 关联的 Insight（I0XX）
> **创建：** YYYY-MM-DD | **负责人：** TIX1号
> **前置：** 前置任务（可选）

## 目标

[一句话：完成什么]

## 架构/方案概述

[核心方案描述，架构图可用文本标识]

## 操作指南（可选）

[启动步骤、常用命令、测试方法]

## 已知问题

1. [问题描述 — 影响 — 当前状态/处理思路]

## 关联

- I0XX: [关联 Insight 说明]
- T0XX: [关联任务说明]
```

### board.md

```markdown
# T0XX board — 任务名称

> **状态：** ⬜ todo｜🟡 doing｜🔵 verifying｜✅ done｜⏸️ archived｜❌ cancelled
> **更新：** YYYY-MM-DD

## 子任务

| # | 任务 | 状态 | 产出/备注 |
|---|------|------|-----------|
| X.1 | ... | ✅ done | `files/xxx` |
| X.2 | ... | 🟡 doing | ... |
| X.3 | ... | ⬜ todo | ... |

## 当前进展

[最近做了什么、遇到什么、下一步计划]

## 里程碑

[关键时间节点]
```

### memory.md

```markdown
# T0XX memory — 任务名称

> 开始：YYYY-MM-DD

## YYYY-MM-DD

- [尝试了什么 + 结果]
- [为什么失败 + 教训]
```

## 核心联动规则

| # | 规则 |
|---|------|
| 1 | **目录里有什么，看板就列什么。** 新增 T0XX → 立刻更新 `board.md` |
| 2 | **总看板状态 = 各 T0XX/board.md 的汇总。** 进度唯一来源是子任务的 board.md |
| 3 | **T0XX 目录只创建一次，不删除不移位。** 归档/取消只改 board.md 状态 |
| 4 | **总看板必须包含全部任务，包括归档的** |
| 5 | **关联双向同步：A 引用了 B，就去 B 的关联里加上 A。** 包括 plan.md/board.md/memory.md/04_tasks/board.md/Insight 所有文档的 `## 关联` 区。 |
| 6 🎯 | **写文档是完成的一部分。** 完成子任务/里程碑后必须**立即**更新 board.md + memory.md + 关联。文档没更新 = 任务没做完。 |
