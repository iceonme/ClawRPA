## 1. 运行架构：专用常驻浏览器 (Resident Chrome)
为了解决登录态复用和设备指纹稳定性，推荐采用“专用浏览器 + CDP 接入”的架构，而非每次脚本启动时新建 Context。

- **核心思路**：
    - 启动一个专门用于自动化任务的 Chrome 实例，拥有独立的 `user-data-dir`。
    - 暴露远程调试端口（CDP Port，如 9222）。
    - 脚本通过 `connect_over_cdp` 接入，直接在已有环境中执行。
- **优势**：极难被识别为自动化工具，因为环境特征与真人日常使用完全一致。

## 2. 交互细化：三层模拟策略

### 第一层：节奏层 (Pacing Layer) - 解决“频率”问题
最基础也最有效的抗风控手段。
- **随机停顿 (Dwell Time)**：进入页面、点击链接、切换 Tab 后，加入 2-5 秒的随机视觉确认停顿。
- **非固定节拍**：所有 `sleep` 或 `delay` 必须使用随机函数（如：`random.uniform(min, max)`）。
- **稳定性门哨 (Stability Gates)**：页面没完全加载（或特定 API 没返回）之前，严禁任何交互。

### 第二层：行为链层 (Chain Layer) - 解决“逻辑”问题
模拟真实的业务探索路径，避免“跳跃式”操作。
- **完整链路**：由“首页 -> 搜索 -> 列表点击 -> 内容阅读 -> 返回/关闭”组成的连贯链路。
- **干扰行为**：在主线任务中加入少量无意义的滑屏、Hover 动作，增加行为熵。

### 第三层：物理层 (Physical Layer) - 解决“特征”问题
在最底层的 API 调用上进行伪装。

#### A. 鼠标运动 (Bezier Trajectory)
- **非线性移动**：通过贝塞尔曲线算法生成坐标序列，调用 `mouse.move` 实现。
- **位置偏置**：点击按钮时，坐标落在中心点以外 80% 范围的可点击区域内，并带有微小抖动（Jitter）。

#### B. 键盘输入 (Typing Dynamics)
- **动态延迟**：模拟真人打字，按键间隔符合正态分布。
- **退格纠错**：偶尔模拟输入错误后按 Backspace 修正，这是极强的人类特征。

## 3. 核心工具函数设计 (参考实现)

```python
# 示例：正态分布打字
async def human_type(page, selector, text):
    await page.focus(selector)
    for char in text:
        # random.gauss(均值, 标准差)
        delay = random.gauss(0.15, 0.05) 
        delay = max(0.05, min(delay, 0.5))
        await page.keyboard.type(char)
        await asyncio.sleep(delay)

# 示例：悬停后点击
async def human_click(page, selector):
    element = page.locator(selector)
    # 获取边界并随机选择点击点
    box = await element.bounding_box()
    tx = box['x'] + box['width'] * random.uniform(0.1, 0.9)
    ty = box['y'] + box['height'] * random.uniform(0.1, 0.9)
    # 先移动并悬停
    await page.mouse.move(tx, ty, steps=random.randint(10, 20))
    await asyncio.sleep(random.uniform(0.2, 0.8))
    await page.mouse.click(tx, ty)
```
