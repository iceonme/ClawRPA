# K002_票牛 H5 平台自动化稳定性与风控绕过实战总结_20260507

## 1. 风控环境绕过 (Context Warm-up)
- **痛点**：直接通过 URL 跳转搜索页或详情页，容易触发“白屏”或“访问异常”，疑似 Referer 检查或 Cookie 缺失。
- **对策**：**首页热身法**。先 `goto` 进入域名根首页，静默等待 10-15 秒，让浏览器完成基础静态资源加载和 Session 建立。
- **跳转技巧**：避免使用 `page.goto` 进行跨域跳转，改用 `page.evaluate("window.location.href = '...'")`。这种方式在前端日志中更像是一次“内部导航”。

## 2. 详情页入口穿透 (Pierce Strategy)
- **痛点**：H5 页面的列表卡片常有透明遮罩或广告弹窗拦截点击事件，导致常规 `click()` 无效。
- **对策**：**三点狙击与强力除障**。
    - 在点击前，用 JS 物理删除页面上所有 `.mask`, `.overlay`, `.download-banner` 节点。
    - **多点狙击**：同时尝试点击卡片内的 `img` (封面)、`.name` (标题) 和 `.price` (价格块)，任意一点触发即可完成跳转。

## 3. 面板交互唤起 (Coordinate Trigger)
- **痛点**：底部的“立即购买”红条按钮由于层级原因，JS 点击（`.click()`）有时无法触发弹窗。
- **对策**：**坐标物理重击**。通过 `getBoundingClientRect()` 获取按钮中心在视口中的绝对坐标，使用 `page.mouse.click(x, y)` 模拟真实的物理触碰。

## 4. 垂直收割嗅探 (Carpet Sniffing)
- **痛点**：不同项目的 SKU 展现形式不同，有的分档位，有的直列，类名（Class）极不稳定。
- **对策**：**地毯式面值识别**。
    - 不再依赖 Class 查找，而是遍历所有文本节点。
    - 使用正则表达式识别特征文本：`^\\d+$` (纯数字面值) 或 `.*元$` (带单位面值)。
    - 排除干扰：利用正则排除“日期格式”（如 `05.16`）和“起”字。

## 5. 容错采集流 (Robust Pipeline)
- **逻辑**：三级嵌套采集（日期 -> 面值 -> 票源）必须配合 `try-except` 块。
- **技巧**：在循环点击面值档位时，若某档位点击失败或不可见，应记录 Log 并跳过，继续采集下一档，确保数据的“最大完备性”。
