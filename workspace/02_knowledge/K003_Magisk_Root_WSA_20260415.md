# K003_Root_WSA_KernelSU_Magisk_20260415

## 1. 什么是 Magisk / KernelSU？
- **Magisk**：传统的 ROOT 授权工具，通过修补 boot.img 实现。
- **KernelSU**：较新的 ROOT 方案，直接在内核层面实现（本项目下载的版本即包含此方案）。它比 Magisk 更难被 App 检测。

## 2. 为什么需要管 Magisk？
对于“云手机”和“RPA (机器人流程自动化)”项目，ROOT 权限至关重要：
- **低级控制**：允许脚本绕过某些应用限制，进行键盘鼠标模拟、屏幕截取。
- **系统修改**：可以安装 LSPosed 等模块，实现对 App 行为的动态拦截和修改。
- **环境伪装**：通过 Magisk 可以修改机型、序列号等信息，防止被 App 检测为虚拟机。

## 3. 安装 WSABuilds 后的核心动作
1. **解压与注册**：解压下载的 `.7z` 包到固定目录（不可删除），运行 `Run.bat`。
2. **如果是 KernelSU 版本**：在 Windows 开始菜单搜索 **KernelSU** 并打开。
3. **如果是 Magisk 版本**：在 Windows 开始菜单搜索 **Magisk** 并打开。
4. **初始化环境**：首次打开可能会提示环境修复，按提示操作。
5. **管理超级用户**：KernelSU 的默认授权方式可能更隐蔽，需在 KernelSU App 内手动为特定 App 勾选“开启超级用户”。

## 4. 进阶配置（按需）
- **Zygisk**：在 Magisk 设置中开启 Zygisk，可以安装更多强大的功能模块。
- **模块安装**：可以通过 Magisk 的“从本地安装”功能加载 `.zip` 格式的安卓增强模块。
