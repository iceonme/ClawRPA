# K004_Tencent_WSA_Analysis_20260415

## 1. 背景调研
腾讯应用宝（PC 版）集成的安卓环境通常被称为“腾讯移动应用引擎”。虽然它利用了一些 Windows 虚拟化技术，但与微软官方的 WSA 或开源社区的 WSABuilds 有显著区别。

## 2. ADB 连通性测试
- **扫描结果**：探测到本地端口 `5555` 处于 `LISTENING` 状态。
- **连接测试**：尝试使用 `adb connect 127.0.0.1:5555` 连接，但目前握手失败。
- **结论**：腾讯版 WSA 对外部调试（ADB）有更严格的封装或非标处理，不建议首选作为 RPA 开发环境。

## 3. 核心对比：应用宝 vs. 开源版 WSABuilds

| 特性 | 应用宝版 (Tencent Engine) | 开源版 (WSABuilds) |
| :--- | :--- | :--- |
| **底层内核** | 腾讯自研定制版 | 微软原生内核 + 社区优化 |
| **GApps 支持** | 部分预装，相对封闭 | 完美支持，可自由定制 |
| **ROOT 权限** | **极难获取** (无公开方案) | **完美支持** (预装 Magisk/KernelSU) |
| **ADB 连通性** | 端口非标、调试开关隐蔽 | 标准端口、一键开启开发者模式 |
| **RPA 适配度** | 低 (适合普通用户运行游戏) | **极高** (专门为开发者和云手机优化) |

## 4. 结论与建议
对于本项目（云手机 RPA），**强烈建议坚持使用开源版 WSABuilds**。
- **理由**：RPA 核心依赖的 ADB 自动化和 Magisk/KernelSU ROOT 授权在应用宝版上极难实现且不稳定。
- **现状**：目前开源版的 10061/593 闪退问题可通过配置 CFG 策略和显卡首选项解决，收益高于切换到更封闭的应用宝。
