# 判断规则

## 目标
找出**值得后续触达**的微博用户，而不是泛泛筛出所有提到演唱会的人。

## 建议标签
- `明确求票`
- `高意向想看`
- `疑似求票`
- `转票 / 出票`
- `普通讨论`
- `无关噪音`

## 高价值信号
以下表达通常值得保留进潜客池：
- 明确表达 `求票 / 收票 / 蹲票 / 还有票吗 / 求两张 / 求连坐`
- 明确表达 `想去但是没抢到 / 没票了 / 抢不到 / 有没有人转`
- 明确问价、问渠道、问场次、问座位
- 在高相关帖子下连续追问购票信息

## 排除信号
以下通常不应进入潜客池：
- 单纯讨论歌单、舞美、情绪感受
- 单纯晒票、晒现场但无购票需求
- 明显是黄牛广告、无关引流、抽奖、营销号
- 明显是 `出票` 而不是 `求票`（可单独保留到卖家池，但不算本任务主目标）

## 建议触达方式
- `私信优先`：用户有明确需求，且主页/uid 可稳定定位
- `评论回复优先`：评论可见但 uid/主页不稳定，或更适合在原帖互动
- `跳过`：信号弱、风险高、疑似营销号、疑似黄牛

## 需要 agent 介入的场景
以下情况建议标记 `needs_agent_review=true`：
- 语义模糊：像想看，又像只是感叹
- 反讽 / 玩笑 / 缩写黑话较多
- 上下文依赖强，需要结合整帖和上下楼回复判断
- 同一用户多条发言互相矛盾
- 无法判断是求票还是出票

## 初步规则
- 明确表达“求票 / 收票 / 求两张 / 还有吗 / 蹲票”等，可判为 `明确求票`
- 表达“想去、想看、没抢到、还有机会吗”等，但未直接说求票，可判为 `高意向想看` 或 `疑似求票`
- 仅讨论演出内容、歌单、情绪表达，不直接视为潜在客户
- 无法确定时优先标 `needs_agent_review=true`，不要硬判高意向
- 建议采用 **规则初筛 + agent 复审** 的组合方式
