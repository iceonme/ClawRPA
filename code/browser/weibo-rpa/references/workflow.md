# 工作流

## 当前微博版标准流程（lead v1）
1. 接收演出查询短语（如 `时代少年团 广州`）
2. **先直接搜索核心关键词本身**，如 `郑州 汪苏泷`，优先拿最大相关帖子池
3. 在搜索结果首页及后续分页中，按两个核心信号筛选候选帖子：**时间**、**回复数/评论数**
4. **优先处理时间更近、回复数更高的帖子**，不再把搜索页浅层评论作为主路径
5. 对高价值候选帖子直接点击进入**帖子主页 / 详情页**
6. 在帖子主页中继续滚动评论区，查看更完整的评论列表，并从评论区抽取 lead
7. 在当前搜索结果页的候选帖子中重复上述过程，直到本页高价值帖子处理完成
8. 再点击搜索结果页底端**下一页**，继续循环，直到当前关键词下已没有高价值帖子（主要看时效性和评论数）
9. 当前关键词完成后，再进入下一个扩展关键词做补召回
10. 进行规则初筛、全局去重与结果输出
11. 在翻页 / 点击帖子 / 进入详情页 / 滚动评论区时，加入短暂停顿与小幅动作，控制访问节奏，更接近真人操作

> 当前版本的主路径应理解为：**核心关键词搜索 → 按时间/回复数筛选高价值帖子 → 直接进帖子主页抓完整评论 → 翻页继续覆盖 → 扩展关键词补召回 → 去重输出**。
> 搜索结果页浅层评论最多只可作为辅助观察，不应再作为主要采集路径。

## 当前推荐拆分
- 查询解析与主查询构造
- 搜索结果页滚动与帖子池扩展
- 搜索页评论展开与查看更多评论
- 评论层 / 正文信号联合抽取
- 规则初筛流程
- 扩展关键词补召回
- 结果输出流程
